About me ...

My name is Carlos Andres Sepúlveda Sanchez (I'm latin guy LOL), I was born in Cúcuta, Colombia but currently I live in Bogotá, Colombia ... I love the basketball and I am becoming an amateur runner.

I grew up, studied and ....

I began my professional life working in the thesis for get the degree as Software Engineer. I worked there designing and implementing a web app that allows to write and compile code online basically. I fell in love of that project, and started to add functionalities like : concurrent editing, chat, and deployment on the browser. In this way I learned technologies like : nodejs, socket.IO, sharejs (An implementation of Operational Transformation), redis; also I had to write a lot of javascript lines. When the project was finished, the app allowed to write code concurrently with a "co-worker" just like google docs.

After I moved to Bogotá, Colombia and began to work in a project that was basically a book reader online, like amazon reader but with more graphics interactions because it was designed for children. In that project I was the lead engineer; We used NodeJs as server side technology, mongoDB as main database and backboneJS as javascript framework. It was a project with many challenges in client side due to the GUI had a lot of functionalities and for that we had to optimize the loading time, render time, and persisting data in the client (SessionStorage, LocalStorage, WebDB).

Then I changed job, started to work in Elibom (I still am working there). This company offer a REST API to send SMS to mobile phones, its main customers are financial companies that send notification messages. I belong to development team and am in charge of designing, improving and implementing the architecture that support the business core. We have a cloud platform (specifically AWS) and it has a concurrent and distributed architecture, to achieve it we use technologies like : Redis, RabbitMQ, Cache System, Stateless, Continous Integration, Continous Deploying, Jenkins, S3, cloudfront.

In Elibom I have had to develop libraries and others functionalities different to business core, for example : a PHP Elibom client, a C# Elibom client, bash scripts for automation, python scripts for automation, lua scripts for redis process. I face any challenge regardless of complexity or code language.

Also I am an entrepreneur, have been trying to found a startup. My first attempt was www.atartel.com, the idea is to offer a REST API to make phone calls automatically (like twilio). There I learned to use Asterisk, Freeswitch and other issues related to VoIP (I did not have idea). The Atartel architecture is distributed and scalable to support any amount of calls, if the queues are growing up, you only need to start a couple of new instances (virtual machines) and the problem is solved. The other attempt is www.facturala.co (It's under construction), the idea is to offer an API to create invoices (similar to freshbooks.com).

Finally, I just want to say, I love learning, it's amazing when you start a new project and do not know many issues (that force you to learn). I am working in personal projects like my website, a micro framework web (https://github.com/carlossepulveda/pongee), rewriting the thesis and home automation (https://github.com/carlossepulveda/home-automation).
